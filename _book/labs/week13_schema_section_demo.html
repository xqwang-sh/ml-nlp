7  Lab Week 13：MinerU、Section、Schema 与 Evidence 示范

7.1 Lab 目标

本 lab 演示 Week 13 的最小抽取链路：

真实巨潮 PDF
  -> MinerU API 转 Markdown
  -> parsed_docs.jsonl
  -> section routing
  -> rule baseline 或硅基流动 LLM 抽取
  -> Pydantic validation

课堂上先讲 MinerU 解析，再讲 section 与 schema，最后再讲字段抽取。字段抽取可以先用规则 baseline，让学生看懂字段、证据和校验；正式项目也可以用硅基流动 API 做 LLM 抽取，但输出仍应通过同一个 Pydantic schema。

7.2 1. 用 MinerU 把 PDF 转 Markdown

运行：

python3 src/parse_docs.py

输出：

data/parsed/markdown/*.md
data/parsed/parsed_docs.jsonl

正式项目中，这一步使用 MinerU API 把真实巨潮 PDF 转为 Markdown，并整理成后续脚本统一读取的结构：每个文档包含 doc_id、title、pdf_path、markdown_path 和 pages。

没有真实 MinerU Key、MinerU 返回失败、或解析结果为空时，脚本会直接退出。这样可以防止学生误把本地解析或空结果当成正式项目结果。

7.3 2. Section Routing

查看规则：

cat configs/section_rules.yaml

运行：

python3 src/route_sections.py

输出：

data/parsed/sections.jsonl
outputs/reports/section_check_report.csv

课堂讲解重点：

  • Routing 只是“自动找候选章节”；
  • Checking 是人工检查候选章节是否真的正确；
  • 目录、释义、备查文件等位置可能误命中；
  • quality_issue 应记录 ok、not_found、too_short、wrong_section 等状态。

7.4 3. Schema

查看：

cat src/schemas.py

本 demo 的核心 schema 是：

class AnnualReportRiskExtract(BaseModel):
    doc_id: str
    company_name: str
    report_year: Literal["2024"]
    event_type: Literal["年报风险披露"]
    risk_categories: list[RiskCategory]

课堂讲解重点：

  • Optional 表示允许缺失；
  • Literal 用来限制枚举值；
  • evidence.text 必须来自原文；
  • 不确定时输出 null，不能编造。

7.5 4. 抽取与校验

运行：

python3 src/extract_fields.py --method rule
python3 src/validate_results.py

输出：

outputs/results/extract_results.jsonl
outputs/results/records_validated.csv
outputs/logs/validation_errors.jsonl

本 demo 使用简单规则抽取，因此结果不会完美。课堂上应特别观察错误字段，把它作为“为什么需要 LLM + evidence + 人工评估”的例子。正式项目可以把 extract_fields.py 替换为 LLM API 调用，但输出仍应通过同一个 Pydantic schema。

如果使用硅基流动 API，在 .env 中填写：

LLM_BASE_URL=https://api.siliconflow.cn/v1
LLM_MODEL=Qwen/Qwen3-8B
LLM_API_KEY=your_key_here

然后运行：

python3 src/extract_fields.py --method llm
python3 src/validate_results.py

7.6 5. 给学生的观察任务

请学生打开一条 records_validated.csv，追溯：

records_validated.csv
  -> extract_results.jsonl
  -> sections.jsonl
  -> parsed_docs.jsonl
  -> data/pdf/*.pdf
  -> metadata.csv

回答：

  1. 哪个字段最容易抽错？
  2. evidence 是否真的出现在原文里？
  3. 如果模型无法判断某个具体风险类别，应该输出什么？
  4. 如果 section 命中错误，会对抽取结果造成什么影响？
