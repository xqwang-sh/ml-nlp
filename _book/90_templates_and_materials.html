12.12 AGENTS.md / CODING_RULES.md

# Agent Instructions for CNINFO Project

## Project Goal
本项目从巨潮资讯网公告 PDF 中抽取结构化金融事件信息。

## Data Rules
- 主数据必须来自巨潮资讯网公开公告。
- 不绕过登录、验证码或访问限制。
- 原始 PDF 不得修改。

## Secret Rules
- 不要把 API Key 写入代码。
- 不要提交 .env。
- 只维护 .env.example。

## Coding Rules
- 每个脚本必须支持命令行运行。
- 所有输出写入 outputs/。
- 所有错误写入 outputs/logs/。
- 修改代码后必须说明运行哪个命令验证。

## Extraction Rules
- 字段值必须来自公告文本。
- 每个关键字段要有 evidence_text。
- 无法判断时输出 null。
- 抽取结果必须通过 Pydantic 校验。

12.21 Starter Code 建议

教师可以发轻量 starter，而不是完整答案。当前 starter_example/ 默认离线运行，不访问真实网站，不调用真实 LLM API。它先让学生看懂项目链路，再去替换真实巨潮数据和 API。

starter_example/
  README.md
  run.py
  .env.example
  requirements.txt
  configs/
    workflow.yaml
    model_config.yaml
    section_rules.yaml
  data/
    metadata/metadata.csv
    parsed/parsed_docs_sample.jsonl
  src/
    common.py
    crawl_cninfo.py
    download_pdfs.py
    check_dataset.py
    llm_client.py
    hello_llm.py
    schemas.py
    parse_with_mineru.py
    route_sections.py
    extract_fields.py
    validate_results.py
    report_results.py
  pipeline_run.py

推荐学生先运行：

cd starter_example
python3 -m pip install -r requirements.txt
python3 run.py all

若需要检查真实 LLM API，再复制 .env.example 为 .env，填入真实 key 后运行：

python3 src/hello_llm.py --config configs/model_config.yaml

Starter code 不应包含真实 API Key，也不应鼓励绕过访问限制。
